# Swatch — Design Context

## Project

**Swatch** is a design system registry — a visual marketplace where people browse, fork, customize, and publish design systems. Built with Next.js 16 App Router, React 19, Tailwind v4, TypeScript.

## Design Context

### Users

Web designers who build applications (web, mobile, anything). They arrive at Swatch looking for a design system they can clone as a head start, then customize with Claude Code for their own project. They live in Linear, Vercel, Notion, Framer, Arc — they have strong expectations for how a professional SaaS tool should look and feel.

### Brand Personality

**Confident. Functional. Quietly distinctive.**

Swatch should feel like a **real tool**, not a portfolio piece. It earns its distinctiveness through *craft*, not through ornament — the quality of spacing, the restraint of the color palette, the smoothness of the interactions. A designer should look at Swatch and think *"this team knows what they're doing,"* not *"cool magazine layout."*

### Aesthetic Direction

**Modern SaaS / Linear-like.**

Reference mood: **Linear**, **Vercel**, **Mobbin**, **Raycast**, **Arc Browser**, **Notion**. A product that feels like a daily driver.

Core moves:
- **Clean modern sans** for everything. Geist Sans is the primary font. No serifs. No italic display type.
- **Dense, functional information.** Information-rich pages. Real stats, real counts, real data. Nothing is for decoration.
- **Subtle borders, small radii.** 1px borders in low-contrast neutrals. Border radius 6-8px, not 16px+ marketing blob shapes.
- **Cool dark palette** with a single saturated accent. OKLCH. Neutrals lean slightly cool. Dark-first.
- **Sharp, crisp cards in dense grids** — not magazine rows with rule lines. Cards ARE allowed here because real tools use them.
- **Hover states are fast and subtle.** Borders shift, backgrounds lift one step. No big shadow pops.
- **Small to medium type.** 13-14px body, 11-12px meta labels, 16-20px subheads, 24-48px display. Never bigger than ~56px except for marketing hero.

Anti-references (what Swatch should NEVER look like):
- Magazine / editorial layouts
- Serif display type (Bodoni, Fraunces, etc.)
- Rule lines as primary dividers
- Massive asymmetric hero typography (clamp(72px, 11vw, 180px))
- Warm editorial red accents
- Italic accent words in headlines
- Dot-grid backgrounds
- Nothing Phone / Teenage Engineering styling
- Glassmorphism decoration
- Purple-to-blue gradients on dark

### Theme

**Dark-first with light toggle.** Linear-style dark (cool near-black, not warm ink). Light mode is fully considered, matches the Vercel/Linear light style.

### Design Principles

1. **Functional over decorative.** If an element doesn't serve the task, remove it. Every pixel should earn its place by doing something, not by looking pretty.
2. **Dense is honest.** Don't pad content into emptiness to make a page look "clean." Real tools show real data.
3. **Distinctive through restraint.** One accent color. One font. Subtle radii. The difference between Linear and its clones is in the details, not the ideas.
4. **Fast and crisp.** Interactions should feel instant. Hover states in 120ms, not 400ms. No bouncy easings.
5. **Dark first, light fully considered.** Neither theme is an afterthought.

### Font Selection

- **Primary**: **Geist Sans** (via next/font/google). Geist is not on the impeccable reflex-fonts-to-reject list, is modern without being overused like Inter, and reads as "Linear-adjacent without being a clone."
- **Monospace** (for code blocks and tertiary labels): **Geist Mono**
- **No display serif.**
- **No italic display type.**

### Accent Color

A single saturated indigo/violet, OKLCH-defined. Default: `oklch(0.62 0.22 280)` — a bright, modern indigo. Used sparingly on primary CTAs, active tab underlines, brand dots. **NEVER** used for body text, borders of "normal" cards, or decoration.
